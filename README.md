# Library — Django Web Application

A Django web application for managing a book library. Built as a school assessment to demonstrate Django models, admin interface, views, and templates — with all code generated using AI.

## Features

- **Book catalog** — browse all books, filter by genre, view details
- **Author pages** — view author info and all their books
- **Admin panel** — full CRUD for books, authors, and genres with search and filters
- **Sample data** — pre-loaded with classic books to demonstrate functionality

## Data Models

The app uses three interconnected models defined in `library/models.py`:

| Model | Fields | Relationships |
|-------|--------|---------------|
| **Author** | first_name, last_name, birth_date, bio | Has many Books |
| **Genre** | name | Many-to-many with Books |
| **Book** | title, published_date, isbn, summary, available, created_at | Belongs to Author, has many Genres |

## Setup Instructions

### Prerequisites

- Python 3.10+

### Installation

```bash
# Clone the repository
git clone <repo-url>
cd hanna_django

# Create and activate a virtual environment
python3 -m venv venv
source venv/bin/activate

# Install Django
pip install django

# Apply database migrations
python3 manage.py migrate

# Create a superuser for the admin panel
python3 manage.py createsuperuser

# Start the development server
python3 manage.py runserver
```

### URLs

| URL | Description |
|-----|-------------|
| http://127.0.0.1:8000/ | Homepage — list of all books |
| http://127.0.0.1:8000/book/1/ | Book detail page |
| http://127.0.0.1:8000/author/1/ | Author detail page |
| http://127.0.0.1:8000/admin/ | Django admin panel |

## How the Code Was Generated

All application code was generated by AI (Claude) in a single conversation:

1. **Models** (`models.py`) — described the data structure (books, authors, genres) and AI generated the Django model classes with appropriate field types, relationships, and meta options.
2. **Admin** (`admin.py`) — AI generated admin registrations with `list_display`, `list_filter`, `search_fields`, and `filter_horizontal` for a polished admin experience.
3. **Views** (`views.py`) — AI created three function-based views with optimized querysets (`select_related`, `prefetch_related`) and genre filtering.
4. **Templates** — AI generated four HTML templates with a responsive CSS design, genre filter badges, and availability indicators.
5. **URLs** — AI set up the URL routing in both the project and app level.
6. **Sample data** — AI generated a script to populate the database with 5 books, 3 authors, and 5 genres.

The entire process from empty directory to working application took approximately 5 minutes. The manual steps were: adding `'library'` to `INSTALLED_APPS` in `settings.py`, and running the Django management commands (`makemigrations`, `migrate`, `createsuperuser`, `runserver`).

## Assessment Reflection

Using AI to build a Django application was remarkably fast and easy. AI excels at generating boilerplate code — model definitions, admin configuration, URL patterns, and template structure — which are the most repetitive parts of Django development. The developer still needs to understand what each piece does, decide on the data structure, and know how to run the Django management commands. AI is a powerful tool that speeds up development, but understanding the fundamentals is still essential.
